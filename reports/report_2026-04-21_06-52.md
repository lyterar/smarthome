### Room3DView.java (добавлен Canvas-прицел)

- **Причина:** в FPS без прицела сложно ориентироваться — нет визуального центра  
  **Следствие:** `Canvas` поверх `SubScene` (в `Pane`), `mouseTransparent=true`, крест `+` с окантовкой  
  **Файл:** `view/component/Room3DView.java`

  **Файл:** `view/component/Room3DView.java`

## GoF паттерн

**Strategy** → `FpsCameraController` / `CameraController` → обоснование: оба контроллера подключаются
через одинаковый интерфейс `attach(SubScene)` / `detach(SubScene)`, Room3DView переключает их
без изменения собственной логики — это классическая реализация паттерна Strategy (GoF, глава 5).

## Технические решения

| Задача | Подход | Причина |
|--------|--------|---------|
| Инерция | lerp с адаптивным коэф. | Линейная интерполяция = O(1), нет численной нестабильности |
| Коллизии | AABB clamp | Достаточно для прямоугольной комнаты, zero overhead |
| Прицел | Canvas 2D overlay | SubScene не поддерживает 2D-элементы напрямую |
| Боббинг | sin(time * freq) * amp | Простой периодический сигнал = реалистичная качка |
| dt | (now - lastNano) / 1e9 | Независимость от частоты кадров |
