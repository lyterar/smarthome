## Изменения (причина → следствие)

- **Причина:** Все действия (toggle, появление комнаты, выбор) происходят мгновенно — нет обратной связи.  
  **Следствие:** Создан `AnimationFactory` с 5 фабричными методами. Вызывающий код вызывает `.play()` — не знает деталей.  
  **Файл:** `view/component/AnimationFactory.java`

- **Причина:** Toggle устройства незаметен в 3D.  
  **Следствие:** `ScaleTransition` (пульс 1.0→0.72→1.0) + `Timeline` для opacity; вызывается из `onToggleDevice()` через `Room3DView.playToggleAnimation()`.  
  **Файлы:** `AnimationFactory.toggleDevice()`, `Room3DView.playToggleAnimation()`, `MainController.onToggleDevice()`

- **Причина:** Комнаты появляются мгновенно при `drawHouse()` — резкий скачок.  
  **Следствие:** `FadeTransition` 0→1 за 450мс с задержкой `col * 80мс` — эффект «волны» появления.  
  **Файл:** `Room3DView.drawHouse()`

- **Причина:** Устройства выглядят не интерактивно — нет реакции на курсор.  
  **Следствие:** `AnimationFactory.attachHoverAnimation()` навешивает `ScaleTransition` 1.0→1.15 при hover — тактильный feedback.  
  **Файл:** `Room3DView.setupDeviceHover()`

- **Причина:** Переход orbit↔FPS резкий — камера прыгает.  
  **Следствие:** `TranslateTransition` 500мс плавно перемещает `orbitCameraGroup` в/из позиции комнаты.  
  **Файл:** `Room3DView.enterFpsMode()`, `Room3DView.exitFpsMode()`

- **Причина:** `playToggleAnimation()` не может найти Group по deviceId без маппинга.  
  **Следствие:** В `Room3DModel` добавлен `Map<String, Group> deviceModelById`; метод `getDeviceModelById(String)` — O(1) поиск.  
  **Файл:** `view/component/Room3DModel.java`

## GoF паттерн

**Factory Method** → `AnimationFactory` — статическая фабрика анимаций.  
Каждый метод (`toggleDevice`, `roomAppear`, `attachHoverAnimation`, `cameraTransition`, `roomHighlight`) возвращает конкретный подтип `Animation`. Вызывающий код работает только с `Animation` — не зависит от конкретного типа.
